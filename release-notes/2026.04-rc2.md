# Release Notes

This is the second release candidate for the fist Canopy BMC release.

The release focuses on supporting the HPE ProLiant Gen11 platforms and is primarily intended for early adopters and evaluation processes. It addresses issues found during testing of `2026.04-rc1`. Additionally it implements new features.
The listings below give a high-level overview for all changes. Please refer also to the release notes of [`2026.04-rc1`](https://github.com/canopybmc/canopybmc/releases/tag/2026.04-rc1).

## Features
- HPE ProLiant Gen11 CHIF Interface
    - CHIF command dispatch skeleton ([#122](https://github.com/canopybmc/canopybmc/pull/122))
    - CHIF I2C proxy: extract PlatDef from host BIOS SPI flash and route BIOS I2C transactions ([#135](https://github.com/canopybmc/canopybmc/pull/135))
    - CHIF EV storage for persistent BIOS settings ([#123](https://github.com/canopybmc/canopybmc/pull/123))
    - CHIF FirmwareCI tests ([#146](https://github.com/canopybmc/canopybmc/pull/146))
- Git-based version detection for os-release ([#130](https://github.com/canopybmc/canopybmc/pull/130))
- QEMU CI target, with additional parallel QEMU runners ([#105](https://github.com/canopybmc/canopybmc/pull/105), [#117](https://github.com/canopybmc/canopybmc/pull/117))
- CI: remove image manipulation/fixing and use build output directly ([#145](https://github.com/canopybmc/canopybmc/pull/145))

## Bugfixes
- Re-arm boot gate after PGOOD de-assertion so UEFI-initiated power cycles and graceful restarts recover correctly ([#139](https://github.com/canopybmc/canopybmc/pull/139))
- Enable power/reset button passthrough in x86-power-control ([#113](https://github.com/canopybmc/canopybmc/pull/113))
- Serial TX console on HPE Gen11 ([#124](https://github.com/canopybmc/canopybmc/pull/124))
- spi-gxp: support addressed reads with dummy cycles so SFDP discovery works on CS1 (host BIOS flash) ([#134](https://github.com/canopybmc/canopybmc/pull/134))
- smbios-mdr: skip undersized Type 0 tables emitted by AMD SP5 firmware (DL365/DL385) so BIOS version reports correctly
  ([#111](https://github.com/canopybmc/canopybmc/pull/111))
- entity-manager: prevent circular probe/registration loop for decorator objects ([#118](https://github.com/canopybmc/canopybmc/pull/118))
- Correct EV Storage return codes to prevent reboot loop on DL320 ([#147](https://github.com/canopybmc/canopybmc/pull/147))
- Drop CHIF grep from Power Control Test ([#132](https://github.com/canopybmc/canopybmc/pull/132))
- Fix README build instructions and signing-key guidance ([#125](https://github.com/canopybmc/canopybmc/pull/125), closes [#121](https://github.com/canopybmc/canopybmc/issues/121))
