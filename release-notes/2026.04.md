# Release Notes

This is the first stable release of Canopy BMC.

The release focuses on supporting the HPE ProLiant Gen11 platforms. It addresses issues found during testing of `2026.04-rc2`. Additionally it updates the OpenBMC base to a more recent version.
The listings below give a high-level overview for all changes. Please refer also to the release notes of [`2026.04-rc1`](https://github.com/canopybmc/canopybmc/releases/tag/2026.04-rc1) and  [`2026.04-rc2`](https://github.com/canopybmc/canopybmc/releases/tag/2026.04-rc2).

## Features
- HPE ProLiant Gen11 CHIF Interface
    - CHIF Security State message handling ([#149](https://github.com/canopybmc/canopybmc/pull/149))
    - CHIF Serial number and product ID command ([#148](https://github.com/canopybmc/canopybmc/pull/148))
- OpenBMC base update from commit `ad8d54b743` to `4bcd6c002b` ([#140](https://github.com/canopybmc/canopybmc/pull/140))

## Bugfixes
- HID gadget failure after BIOS-to-OS EHCI handoff; rebooting the system will keep the KVM keyboard connected ([#143](https://github.com/canopybmc/canopybmc/pull/143))
- IPMI KCS channel config mismatch ([#154](https://github.com/canopybmc/canopybmc/pull/154))
- WebUI and Patch rebase after OpenBMC update ([#140](https://github.com/canopybmc/canopybmc/pull/140))
- Add missing uboot-sig partition to update tarball ([#140](https://github.com/canopybmc/canopybmc/pull/140))
- Switch to linux-yocto-fitimage kernel recipe as required by the latest OpenBMC upstream revision ([#140](https://github.com/canopybmc/canopybmc/pull/140))
- Add signature to FIT image ([#140](https://github.com/canopybmc/canopybmc/pull/140))
- dropbear: Disable SSH RSA key generation to improve boot time ([#73](https://github.com/canopybmc/canopybmc/pull/73))
- smbios-mdr: Remove cpuinfo service which does not work on GXP platform ([#73](https://github.com/canopybmc/canopybmc/pull/73))
- peci: gxp: prevent overlapping transfers ([#152](https://github.com/canopybmc/canopybmc/pull/152))
- bmcweb: prevent deletion of own user account ([#158](https://github.com/canopybmc/canopybmc/pull/158))
- bmcweb: return proper error on user deletion instead of returning 404 not found every time ([#158](https://github.com/canopybmc/canopybmc/pull/158))
- user-mgr: prevent uid 0 user deletion ([#158](https://github.com/canopybmc/canopybmc/pull/158))
- Remove Arch Linux from the list of supported systems in README ([#156](https://github.com/canopybmc/canopybmc/pull/156))
- Base toolchain now builds with GCC 15 ([#140](https://github.com/canopybmc/canopybmc/pull/140), Closes [#77](https://github.com/canopybmc/canopybmc/issues/77))
